# Руководство пользователя FX-АСУБАНК

Документ описывает порядок работы с веб-интерфейсом подсистемы обработки FX-сделок для трёх ролей: трейдера, позиционера и аудитора.

## 1. Вход в систему

Откройте веб-интерфейс по адресу `http://localhost:5173`. На экране входа выберите роль и введите учётные данные, полученные у администратора. После успешной авторизации откроется дашборд, соответствующий вашей роли. Сессия поддерживается через JWT, который обновляется автоматически при истечении access-токена.

## 2. Работа в роли трейдера

Дашборд трейдера содержит сводку по созданным сделкам, ленту курсов ЦБ и быстрые действия. Для создания новой сделки нажмите "+ Новая сделка" в правом верхнем углу или в боковом меню.

Вводная форма состоит из четырёх разделов. В первом разделе укажите тип сделки (TOD, TOM, SPOT, FORWARD), даты сделки и валютирования, направление операции (Покупка или Продажа) и контрагента. Контрагент выбирается через поле с автодополнением по коду, наименованию или BIC. Во втором разделе укажите валюту покупки, сумму, валюту продажи (сумма продажи рассчитывается автоматически по курсу) и курс. В третьем разделе "Платежи" задайте параметры платёжных операций: валюту, сумму, дату, счёт НОСТРО и направление поступления. В четвёртом разделе при необходимости укажите комментарий.

Кнопка "Сохранить (Редактируется)" сохраняет черновик без перехода в очередь согласования. Кнопка "Отправить досрочно" переводит сделку в статус "Отправлено" и направляет её в очередь позиционера; такое действие требует подтверждения. По достижении регламентного срока сделка переходит в статус "Отправлено" автоматически.

При возврате сделки на исправление откройте её в списке сделок с пометкой "На исправлении". В карточке вы увидите комментарий позиционера. Используйте кнопку "Взять в работу", чтобы вернуть сделку в статус черновика, внесите правки и снова отправьте на согласование.

Для отмены сделки, находящейся в черновике, используйте кнопку "Отменить сделку" в карточке. Действие требует подтверждения и комментария.

## 3. Работа в роли позиционера

Дашборд позиционера выводит счётчики очереди согласования, текущие позиции по валютам и быстрые действия. Перейдите в раздел "Очередь" для работы с поступившими сделками. Сортировка по умолчанию учитывает приоритет регламентного срока.

Сводная форма сделки содержит параметры, реквизиты контрагента, информацию о счетах и чек-лист валидации. В правой колонке выводятся остатки по счетам с учётом планируемой сделки и индикатор покрытия. Доступны три действия: "Согласовать", "Вернуть на исправление" и "Отклонить". При возврате на исправление обязательно указывается причина в комментарии; без комментария действие блокируется.

При согласовании сделка передаётся в систему позиций. При успешной передаче сделка автоматически переходит в статус "Исполнена". В случае ошибки интеграции сделка остаётся в статусе "Согласована" и возвращается код ошибки 502; повторная отправка инициируется администратором.

## 4. Работа в роли аудитора

Дашборд аудитора показывает агрегированные показатели по журналу аудита, ссылки на отчёты и историю изменений. Раздел "Журнал аудита" позволяет применять фильтры по дате, пользователю и типу действия (CREATE, EDIT, SUBMIT, APPROVE, RETURN, REJECT, EXPORT, LOGIN). Журнал доступен в виде таблицы и в выгрузке CSV через кнопку "↓ Экспорт CSV".

В разделе "Отчёты" доступно формирование сводного отчёта за произвольный период. Укажите фильтры по статусу, типу сделки, контрагенту и интервалу дат, выберите формат (JSON, CSV или XLSX) и нажмите "Сформировать". Файл будет загружен автоматически.

## 5. Уведомления и подсказки

Боковая лента курсов обновляется автоматически. Уведомления о событиях системы выводятся в виде всплывающих сообщений в правом нижнем углу и продублированы на странице "Уведомления". Подсказки по полям ввода доступны в форме новой сделки в виде блока "Подсказки".

## 6. Завершение сеанса

Нажмите кнопку выхода в правом нижнем углу боковой панели. Сессия будет завершена; для повторного входа потребуется указать учётные данные.
